MÉMENTO EXAMEN : ALGÈBRE RELATIONNELLE & FONDAMENTAUX SQL

1. Algèbre RelationnelleFormalisme
σ_{cond}(R) Sélection : Filtre les lignes.
π_{A1,A2}(R) Projection : Garde les colonnes.
R ⨝ S Jointure Naturelle : Sur noms communs.
R ⨝_{cond} S Jointure Thêta : Condition R.a = S.b.
ρ_{A→B}(R) Renommage : Change le nom d'un attribut.
Règle d'or : Appliquer σ le plus tôt possible pour réduire la taille des tables avant les jointures.
2. Opérations Ensemblistes∪ ∩ −
R ∪ S Union : Lignes dans R OU dans S.
R ∩ S Intersection : Lignes dans R ET dans S.
R − S Différence : Dans R mais PAS dans S.
Compatible : R et S doivent avoir le même schéma (nombre et types de colonnes).
3. Structure SQL de baseDQL
SELECT [DISTINCT] colonnes
FROM   table1 [AS alias]
JOIN   table2 ON condition
WHERE  filtres (AND/OR/NOT)
ORDER BY col [ASC/DESC]
LIMIT  n OFFSET m;
Ordre d'exécution logique : FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT.
4. Opérateurs de filtrageWHERE
=, <>Égal, Différent
BETWEEN a AND bIntervalle inclusif
IN (v1, v2, ...)Appartenance liste
LIKE 'H%'Flou : % (plusieurs), _ (un)
IS NULLTest de valeur nulle

⚠ NOT IN + NULL : Si la sous-requête contient un NULL, NOT IN ne retournera aucune ligne. Utiliser NOT EXISTS.

5. Jointures AvancéesJoins
INNER JOIN : Intersection stricte (lignes correspondantes).
LEFT JOIN : Tout de la table de gauche + correspondance droite (sinon NULL).
Auto-jointure : Joindre une table avec elle-même.
SELECT e.nom, s.nom AS chef
FROM employe e
JOIN employe s ON e.id_sup = s.id_nss;
6. Schéma de référenceTD1
Employé (NSS, Nom, Salaire, #NoDep, #Supérieur)
Département (NoDep, NomD, #Directeur)
Projet (NoPro, NomP, Lieu, #NoDep)
Travaille (#NSS, #NoPro, Heures)
7. Exemples Algèbre ↔ SQLTraductions
Projection + Sélection : 
π_{Nom}(σ_{Salaire > 3000}(Employé))
SELECT Nom FROM Employé WHERE Salaire > 3000;
Jointure : 
π_{Nom, NomD}(Employé ⨝ Département)
SELECT e.Nom, d.NomD 
FROM Employé e JOIN Département d ON e.NoDep = d.NoDep;
Différence (Employés sans projet) : 
π_{NSS}(Employé) − π_{NSS}(Travaille)
SELECT NSS FROM Employé
WHERE NSS NOT IN (SELECT NSS FROM Travaille);
8. Agrégats de baseStats
COUNT(*), SUM(), AVG(), MIN(), MAX() 
SELECT NoDep, AVG(Salaire)
FROM   Employé
GROUP BY NoDep
HAVING AVG(Salaire) > 4000;
HAVING filtre les groupes (après GROUP BY). WHERE filtre les lignes (avant GROUP BY).

MÉMENTO EXAMEN : SQL AVANCÉ & LOGIQUE

9. Fonctions de FenêtreOVER()

Calculent sur un groupe de lignes sans les fusionner.

fonction() OVER(PARTITION BY... ORDER BY...)
FonctionRésultat ex-aequo
RANK()1, 1, 3 (saute)
DENSE_RANK()1, 1, 2 (suite)
ROW_NUMBER()1, 2, 3 (unique)
-- Salaire cumulé par service
SELECT nom, salaire,
SUM(salaire) OVER(PARTITION BY dep ORDER BY nom)
FROM employes;
10. CTE & Sous-requêtesWITH
CTE (Common Table Expression) : 
WITH Moyennes AS (
  SELECT NoDep, AVG(Salaire) as avg_s
  FROM Employé GROUP BY NoDep
)
SELECT e.Nom 
FROM Employé e JOIN Moyennes m ON e.NoDep = m.NoDep
WHERE e.Salaire > m.avg_s;
Préférer les CTE aux sous-requêtes imbriquées pour la lisibilité.
11. Logique ConditionnelleCASE WHEN
SELECT nom,
CASE 
  WHEN salaire >= 5000 THEN 'Haut'
  WHEN salaire >= 3000 THEN 'Moyen'
  ELSE 'Bas'
END AS categorie
FROM employe;

⚠ Ordre : La première condition vraie gagne. Toujours aller du plus restrictif au plus général.

12. Fonctions SystèmesUtilitaires
Texte :
  • UPPER(s), LOWER(s)
  • CONCAT(s1, s2) ou s1 || s2
  • SUBSTR(s, start, len)
Dates :
  • CURRENT_DATE, NOW()
  • EXTRACT(YEAR FROM date)
  • DATEDIFF / AGE (dépend du SGBD)
Maths :
  • ROUND(v, n), COALESCE(v, 0) (remplace NULL)
13. Pièges ClassiquesAttention
  • AND vs OR : Utiliser des parenthèses si mélange !
  • COUNT(col) vs COUNT(*) : COUNT(col) ignore les NULLs, COUNT(*) compte tout.
  • Jointure oubliée : Produit cartésien (énorme erreur de performance).
  • ALIAS : Obligatoires pour les tables dérivées FROM (SELECT...) AS t.
14. PerformanceIndex
Un index accélère les SELECT ... WHERE et les JOIN, mais ralentit les INSERT/UPDATE.
CREATE INDEX idx_nom ON employe(nom);

Clé Primaire = Index Unique + NOT NULL.

Conseils ExamenMethodo
1. Lire les relations et souligner les clés.
2. Identifier les tables nécessaires.
3. Écrire la jointure AVANT les filtres.
4. Vérifier les doublons (DISTINCT).
5. En algèbre, utiliser des étapes intermédiaires (←).
