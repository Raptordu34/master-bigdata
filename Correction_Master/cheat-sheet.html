Examen 2026

1. Syntaxe & Ordre d'Exécution

SELECT [DISTINCT] col1, AVG(col2)
FROM table_A [JOIN table_B ON A.id = B.id]
WHERE condition_lignes (filtre brut)
GROUP BY col_regroupement
HAVING condition_groupes (filtre post-agrégat)
ORDER BY col [ASC|DESC]
LIMIT n OFFSET m
Ordre réel : FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → ORDER BY.

2. Opérateurs de Filtrage

=, <>, !=, <, >Comparaison standard
BETWEEN a AND bInclusif : [a, b]
IN (v1, v2, ...)Appartenance à une liste
LIKE 'patt%'% (0+), _ (1 seul car.)
IS NULLTest de valeur nulle

3. Agrégats & Groupements

COUNT(*), SUM(), AVG(), MIN(), MAX()

Règle : Toute colonne dans SELECT non agrégée doit figurer dans GROUP BY.

HAVING s'utilise uniquement avec des fonctions d'agrégation.

4. Jointures Standards

  • INNER JOIN : Intersection stricte.
  • LEFT JOIN : Tout A + B correspondant (ou NULL).
  • LEFT JOIN : Tout A + B correspondant (ou NULL).
  • FULL JOIN : Union des deux (avec NULLs).
  • NATURAL JOIN : Jointure auto sur colonnes de même nom (⚠️ DANGEREUX).

Piège : Oublier la clause ON crée un produit cartésien (A × B lignes).

5. Fragmentation (Systèmes Répartis)

Fragmentation Horizontale (FH) :

  • Découpe par lignes via σ (Sélection).
  • Fragments disjoints : R1 ∩ R2 = ∅.
  • Reconstruction : R = R1 ∪ R2 ∪ ... (Union).

Fragmentation Verticale (FV) :

  • Découpe par colonnes via π (Projection).
  • ⚠️ La clé primaire doit être dans chaque fragment.
  • Reconstruction : R = F1 ⋈ F2 ⋈ ... (Jointure).

Propriétés obligatoires :

  • Complétude : Aucune donnée perdue.
  • Reconstructibilité : Opération inverse possible.
  • Disjonction : Pas de duplication (sauf PK en FV).

6. Fragmentation Mixte & Ordre

HV (H puis V) : On filtre par lignes d'abord. On peut ensuite projeter différemment sur chaque groupe de lignes.

VH (V puis H) : On projette d'abord. ⚠️ Si la colonne de filtrage de H est supprimée par V, H devient impossible.

Reconstruction mixte : Reconstituer d'abord les horizontaux (Union), puis joindre le tout.

7. Fonctions Utilitaires

Texte : UPPER(), LOWER(), CONCAT(a, b), SUBSTR(s, start, len), TRIM().

Maths : ROUND(v, d), COALESCE(val, replacement) (gère les NULLs).

Date : NOW(), DATE_PART('year', d), DATEDIFF(end, start).

8. Sous-requêtes & CTE

IN / NOT IN : Test sur ensemble.

EXISTS / NOT EXISTS : Plus performant que IN (s'arrête dès le 1er match).

CTE (WITH) : Lisibilité accrue.

WITH TableTemp AS (
  SELECT ... FROM ...
) SELECT * FROM TableTemp;

9. Logique Conditionnelle (CASE)

SELECT Name,
 CASE 
  WHEN Age < 18 THEN 'Enfant'
  WHEN Age < 65 THEN 'Adulte'
  ELSE 'Senior'
 END as Tranche
FROM Personne;

10. Fonctions de Fenêtrage (Over)

Calculs sans grouper les lignes.

SELECT Dept, Salaire,
 RANK() OVER(PARTITION BY Dept ORDER BY Salaire DESC) as Rang
FROM Employes;
  • PARTITION BY : Définit la fenêtre.
  • ORDER BY : Trie dans la fenêtre.
  • RANK() vs DENSE_RANK() (trous vs pas de trous).

11. Optimisation Coût (CBA)

Modèle de coût : Principalement basé sur les lectures de blocs (I/O).

Taille en blocs : B(R) = ⌈ NbTuples / FacteurBlocage ⌉.

Block Nested Loop Join (BNLJ) :

Coût = B(R) + ⌈ B(R) / M ⌉ × B(S)
  • R : Table externe (chargée par lots).
  • S : Table interne (relue entièrement).
  • M : Taille du Buffer (mémoire vive).

Règle d'or : La table interne est relue B(R)/M fois. Toujours mettre la plus petite table en externe pour minimiser les relectures.

12. Optimisation Logique

Push-down selection : Toujours appliquer σ le plus tôt possible pour réduire la taille des relations avant jointure.

Projection précoce : Supprimer les colonnes inutiles avant les opérations lourdes.

13. Indexation : B-Tree & BJI

B-Tree (Standard) :

  • Efficace pour =, <, >, BETWEEN.
  • Règle du préfixe le plus sélectif : Dans un index composite (A, B), l'ordre doit suivre la probabilité de filtrage.

Binary Join Index (BJI) :

  • Mémorise les liens entre une Dimension (D) et une table de Faits (F).
  • Stocke des paires (D_RID, {F_RIDs}).
Utilisation BJI :
1. Filtre sur Dimension → D_RIDs.
2. Accès BJI → F_RIDs (identifiants faits).
3. Intersection des F_RIDs si multi-index.
4. Lecture physique finale dans F.

14. Vues Matérialisées

Stocke le résultat sur disque. Doit être rafraîchie.

CREATE MATERIALIZED VIEW MV AS
SELECT ... WITH DATA;

Optimise les requêtes répétitives sur gros volumes.
