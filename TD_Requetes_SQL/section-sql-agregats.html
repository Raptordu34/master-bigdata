Niveau 2 — Agrégats et GROUP BY

Intermédiaire ⏱ 35–45 min
Objectifs
  • Maîtriser les cinq fonctions d'agrégation : COUNT, SUM, AVG, MIN, MAX
  • Regrouper des lignes avec GROUP BY et comprendre la règle de cohérence du SELECT
  • Filtrer des groupes avec HAVING et distinguer son rôle de celui de WHERE
  • Combiner jointures et agrégats pour des analyses multi-tables
Rappel de cours — Fonctions d'agrégation
Fonction Rôle Comportement avec NULL
COUNT(*) Compte toutes les lignes du groupe Inclut les NULL
COUNT(col) Compte les valeurs non-nulles d'une colonne Ignore les NULL
SUM(col) Somme des valeurs d'une colonne Ignore les NULL
AVG(col) Moyenne des valeurs non-nulles Ignore les NULL
MIN(col) Valeur minimale du groupe Ignore les NULL
MAX(col) Valeur maximale du groupe Ignore les NULL

Règle fondamentale du SELECT

Toute colonne présente dans SELECT et non agrégée doit obligatoirement figurer dans la clause GROUP BY.

-- OK
SELECT dept_id, COUNT(*)
FROM employes
GROUP BY dept_id;

-- ERREUR : nom non agrégé et absent du GROUP BY
SELECT dept_id, nom, COUNT(*)
FROM employes
GROUP BY dept_id;

Ordre d'exécution logique d'une requête SQL

FROM  →  WHERE  →  GROUP BY  →  HAVING  →  SELECT  →  ORDER BY

WHERE filtre les lignes avant le regroupement.
HAVING filtre les groupes après agrégation — c'est la seule clause où l'on peut conditionner sur un agrégat.

Astuce : Pour arrondir un résultat décimal, utilisez ROUND(AVG(salaire), 2). Le second argument indique le nombre de décimales souhaitées.
Erreur classique : WHERE COUNT(*) > x est syntaxiquement invalide. WHERE s'exécute avant que les groupes n'existent — il ne peut donc pas porter sur un agrégat. Utilisez toujours HAVING COUNT(*) > x pour filtrer après regroupement.
Q1 Fondamental

Comptez le nombre total d'employés dans l'entreprise.

Rappel : COUNT(*) compte toutes les lignes de la table, y compris celles dont certaines colonnes contiennent NULL. C'est la façon standard de compter des enregistrements.
Ctrl+Entrée pour exécuter

Solution — Q1

SELECT COUNT(*) AS nb_employes
FROM employes;

COUNT(*) renvoie ici 15 (le nombre total de lignes de la table). Donner un alias explicite avec AS nb_employes rend le résultat immédiatement lisible.

Q2 Fondamental

En une seule requête, calculez le salaire moyen (arrondi à 2 décimales), le salaire minimum et le salaire maximum de tous les employés.

Astuce : Plusieurs fonctions d'agrégation peuvent coexister dans le même SELECT sans nécessiter de GROUP BY dès lors qu'elles s'appliquent toutes à l'ensemble de la table. Pensez à ROUND(..., 2) pour l'arrondi du salaire moyen.
Ctrl+Entrée pour exécuter

Solution — Q2

SELECT
    ROUND(AVG(salaire), 2) AS salaire_moyen,
    MIN(salaire)           AS salaire_min,
    MAX(salaire)           AS salaire_max
FROM employes;

Les trois fonctions s'appliquent simultanément à l'ensemble des lignes. Sans GROUP BY, la table entière constitue un unique groupe. Le résultat attendu : moyenne autour de 55 000 €, minimum à 38 000 €, maximum à 80 000 €.

Q3 Intermédiaire

Affichez le nombre d'employés par département, en affichant le nom du département (et non son identifiant). Triez par nombre d'employés décroissant.

Piège courant : En joignant deux tables, toute colonne du SELECT non agrégée (ici d.nom) doit figurer dans le GROUP BY. Oublier d.nom dans le GROUP BY provoque une erreur.
Commencez par joindre employes e et departements d sur e.departement_id = d.id, puis sélectionnez d.nom et COUNT(*).
Le GROUP BY doit porter sur d.id, d.nom (ou simplement d.nom si les noms sont uniques). L'ordre final : ORDER BY nb_employes DESC.
Ctrl+Entrée pour exécuter

Solution — Q3

SELECT
    d.nom            AS departement,
    COUNT(*)         AS nb_employes
FROM employes e
JOIN departements d ON e.departement_id = d.id
GROUP BY d.id, d.nom
ORDER BY nb_employes DESC;

Grouper sur d.id, d.nom est une bonne pratique : d.id garantit l'unicité du groupe même si deux départements avaient le même nom, tandis que d.nom est requis car il apparaît dans le SELECT.

Q4 Intermédiaire

Calculez le salaire moyen par département (avec le nom du département). Affichez le résultat trié par salaire moyen décroissant, arrondi à 2 décimales.

Reprenez la structure JOIN de Q3, remplacez COUNT(*) par ROUND(AVG(e.salaire), 2), et triez sur cet alias.
Ctrl+Entrée pour exécuter

Solution — Q4

SELECT
    d.nom                       AS departement,
    ROUND(AVG(e.salaire), 2)    AS salaire_moyen
FROM employes e
JOIN departements d ON e.departement_id = d.id
GROUP BY d.id, d.nom
ORDER BY salaire_moyen DESC;

Il est possible d'utiliser directement l'alias salaire_moyen dans ORDER BY en SQLite. Dans d'autres SGBD (PostgreSQL strict, MySQL ancien), il faut répéter l'expression : ORDER BY ROUND(AVG(e.salaire), 2) DESC.

Q5 Intermédiaire

Affichez uniquement les départements comptant plus de 3 employés, avec le nom du département et le nombre d'employés.

Erreur fréquente : WHERE COUNT(*) > 3 est syntaxiquement invalide — WHERE s'exécute avant le regroupement, les agrégats n'existent donc pas encore. La bonne clause est HAVING COUNT(*) > 3, placée après GROUP BY.
Après GROUP BY d.id, d.nom, ajoutez la ligne HAVING COUNT(*) > 3. Cette clause filtre les groupes dont l'effectif ne dépasse pas le seuil.
Ctrl+Entrée pour exécuter

Solution — Q5

SELECT
    d.nom        AS departement,
    COUNT(*)     AS nb_employes
FROM employes e
JOIN departements d ON e.departement_id = d.id
GROUP BY d.id, d.nom
HAVING COUNT(*) > 3
ORDER BY nb_employes DESC;

WHERE vs HAVING — résumé

  • WHERE salaire > 40000 — élimine les lignes avant le regroupement
  • HAVING COUNT(*) > 3 — élimine les groupes après le regroupement
  • Les deux peuvent coexister dans la même requête
Q6 Intermédiaire

Calculez le total des heures travaillées par projet à partir de la table affectations. Affichez le nom du projet (jointure avec projets) et triez par total d'heures décroissant.

Rappel : La table affectations contient une ligne par couple (employé, projet). La colonne heures indique le nombre d'heures investies par cet employé sur ce projet. SUM(heures) additionne toutes ces contributions pour un même projet.
Joignez affectations a et projets p sur a.projet_id = p.id, groupez par p.id, p.nom, et calculez SUM(a.heures).
Ctrl+Entrée pour exécuter

Solution — Q6

SELECT
    p.nom               AS projet,
    SUM(a.heures)       AS total_heures
FROM affectations a
JOIN projets p ON a.projet_id = p.id
GROUP BY p.id, p.nom
ORDER BY total_heures DESC;

SUM ignore les NULL : si une affectation ne renseigne pas les heures, elle ne contribue pas au total. Pour détecter ce cas, comparez COUNT(*) et COUNT(a.heures) — une différence signale des valeurs manquantes.

Q7 Avancé

Affichez le nombre d'employés par poste, mais uniquement pour les postes dont le salaire moyen dépasse 50 000 €. Triez par salaire moyen décroissant.

Méthode : Cette question combine GROUP BY poste, un calcul de COUNT(*) et de AVG(salaire), puis un filtre HAVING sur l'agrégat de salaire. Deux agrégats différents dans le même groupe.
Commencez par : SELECT poste, COUNT(*) AS nb, AVG(salaire) AS moy FROM employes GROUP BY poste. Ajoutez ensuite le filtre HAVING sur AVG(salaire).
La clause de filtre est HAVING AVG(salaire) > 50000. Vous pouvez afficher les deux agrégats simultanément dans le SELECT.
Ctrl+Entrée pour exécuter

Solution — Q7

SELECT
    poste,
    COUNT(*)                    AS nb_employes,
    ROUND(AVG(salaire), 2)      AS salaire_moyen
FROM employes
GROUP BY poste
HAVING AVG(salaire) > 50000
ORDER BY salaire_moyen DESC;

Notez que le HAVING utilise AVG(salaire) (l'expression complète) et non l'alias salaire_moyen — les alias du SELECT ne sont pas disponibles dans HAVING dans la plupart des SGBD (sauf SQLite qui est plus permissif). Habituer-vous à répéter l'expression dans HAVING pour écrire du SQL portable.

Q8 Avancé

Pour chaque département, affichez en une seule requête : le nom du département, le nombre d'employés, le salaire moyen (arrondi à 2 décimales) et le salaire maximum. Triez par nombre d'employés décroissant.

Conseil : Cette requête synthétise les techniques de Q3, Q4 et Q5 en une seule. Pensez à préfixer chaque colonne par son alias de table (e. ou d.) pour éviter toute ambiguïté, et n'oubliez pas de mettre toutes les colonnes non-agrégées dans le GROUP BY.
Structure : SELECT d.nom, COUNT(*), ROUND(AVG(e.salaire),2), MAX(e.salaire) FROM employes e JOIN departements d ON ... GROUP BY d.id, d.nom ORDER BY ...
Ctrl+Entrée pour exécuter

Solution — Q8

SELECT
    d.nom                       AS departement,
    COUNT(*)                    AS nb_employes,
    ROUND(AVG(e.salaire), 2)    AS salaire_moyen,
    MAX(e.salaire)              AS salaire_max
FROM employes e
JOIN departements d ON e.departement_id = d.id
GROUP BY d.id, d.nom
ORDER BY nb_employes DESC;

Récapitulatif des colonnes

  • d.nom — non agrégée → dans GROUP BY
  • COUNT(*) — agrégée → pas dans GROUP BY
  • ROUND(AVG(e.salaire), 2) — agrégée → pas dans GROUP BY
  • MAX(e.salaire) — agrégée → pas dans GROUP BY
Pour aller plus loin
  • Diversité des postes par département : utilisez COUNT(DISTINCT poste) pour compter le nombre de postes distincts dans chaque département — distinguez ce résultat de COUNT(*) qui compte les employés.
  • Impact des NULL sur AVG : insérez un employé fictif avec salaire = NULL et observez comment AVG(salaire), COUNT(*) et COUNT(salaire) se comportent différemment — une illustration concrète des règles sur les valeurs manquantes.
  • Département avec la plus haute masse salariale : calculez SUM(salaire) par département et limitez le résultat à 1 ligne avec LIMIT 1 après tri décroissant.
  • Projets sans affectations : reprenez Q6 avec un LEFT JOIN entre projets et affectations pour faire apparaître les projets dont le total d'heures est NULL (aucune affectation enregistrée).
  • Double agrégation — employés multi-projets : à partir des tables affectations et employes, calculez pour chaque employé le nombre de projets auxquels il participe (COUNT(DISTINCT projet_id)), puis filtrez avec HAVING ceux qui interviennent sur plus d'un projet.
