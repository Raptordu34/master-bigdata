Mémento SQL — 1. Fondamentaux & Analyse de Données

Examen 2026

1. Structure & Logique d'exécution

Ordre d'écriture (Obligatoire) : 
1. SELECT [DISTINCT] col1, col2
2. FROM table_A [JOIN table_B ON ...]
3. WHERE condition_sur_lignes
4. GROUP BY colonnes_de_regroupement
5. HAVING condition_sur_groupes
6. ORDER BY col [ASC|DESC]
7. LIMIT n OFFSET m
Ordre d'exécution réel (Le moteur SQL) :
1. FROM & JOIN (Choix des données)
2. WHERE (Filtrage brut)
3. GROUP BY (Organisation)
4. HAVING (Filtre après calcul)
5. SELECT (Projection & Alias)
6. ORDER BY (Tri final)

2. Prédicats de Filtrage (WHERE)

TypeSyntaxe & Exemple
Comparaison= , != , < , > , <= , >=
InclusionIN (1, 2, 10) ou IN (SELECT...)
IntervalleBETWEEN 10 AND 20 (Inclusif)
MotifLIKE 'J%' (%:0+ char, _:1 char)
Valeur videIS NULL / IS NOT NULL
Logique Booléenne : AND (prioritaire), OR, NOT.
Ex: WHERE (A OR B) AND C

3. Tri & Pagination

ORDER BY sal DESC, nom ASC; (Tri prioritaire sur sal)
LIMIT 10 OFFSET 20; (Prend 10 lignes à partir de la 21ème).

4. Maîtrise des Jointures (JOIN)

Relier tables A et B via une clé commune (ex: A.fk = B.pk).

INNER JOINLignes avec correspondance dans les deux.
LEFT JOINTout A + B si existe (sinon NULL).
RIGHT JOINTout B + A si existe (sinon NULL).
FULL JOINUnion de LEFT et RIGHT (Toutes les lignes).
CROSS JOINProduit cartésien (Chaque A avec chaque B).

Jointure de N tables :

SELECT t1.a, t2.b, t3.c
FROM t1
JOIN t2 ON t1.id = t2.t1_id
JOIN t3 ON t2.id = t3.t2_id;

Auto-jointure (Self-Join) :

Indispensable pour les relations parent/enfant ou manager/employé. Alias obligatoires.

SELECT e.nom AS Emp, m.nom AS Manager
FROM employes e
LEFT JOIN employes m ON e.manager_id = m.id;

5. Agrégats & GROUP BY

Calculs sur des ensembles de lignes.

FonctionNULL Handling
COUNT(*)Compte absolument tout.
COUNT(col)Ignore les NULL.
AVG(col)Moyenne (Ignore NULL).
SUM / MAX / MINIgnore NULL.
LA RÈGLE D'OR : Si vous utilisez une fonction d'agrégation (SUM, AVG...), toutes les autres colonnes dans le SELECT doivent obligatoirement être listées dans le GROUP BY.

HAVING (Le filtre de groupe) :

S'utilise pour filtrer sur le résultat d'un agrégat.

SELECT ville, COUNT(*)
FROM clients
GROUP BY ville
HAVING COUNT(*) > 5; -- Filtre les villes de + de 5 clients

6. Traitement des NULL

NULL = Valeur inconnue (différent de 0 ou vide).

COALESCE(col, 0) : Retourne 0 si col est NULL.
NULLIF(a, b) : Retourne NULL si a=b.
Attention : Toute opération avec NULL retourne NULL (5 + NULL = NULL).

Examen 2026

7. Sous-requêtes (Subqueries)

1. Scalaire (1 valeur) :

WHERE prix > (SELECT AVG(prix) FROM prod)

2. Liste (1 colonne) :

WHERE id IN (SELECT id_client FROM commandes)

3. Corrélée (Dépendante) :

La sous-requête s'exécute pour chaque ligne de la requête parente.

SELECT nom, sal
FROM emp e
WHERE sal > (
  SELECT AVG(sal) FROM emp 
  WHERE dept_id = e.dept_id
);

4. EXISTS vs IN :

EXISTS est souvent plus rapide pour les gros volumes car il s'arrête dès la première correspondance trouvée.

8. Opérateurs Ensemblistes

Combinaison verticale (même nombre/type de colonnes).

  • UNION : Fusionne + supprime les doublons.
  • UNION ALL : Fusionne tout (Rapide).
  • INTERSECT : Uniquement les lignes présentes dans les deux.
  • EXCEPT : Lignes du 1er absentes du 2ème.

9. CTE (WITH) & Lisibilité

Permet de définir des "tables temporaires" nommées pour simplifier les requêtes imbriquées.

WITH total_ventes AS (
  SELECT vendeur_id, SUM(ca) as ca_total
  FROM ventes GROUP BY vendeur_id
)
SELECT * FROM vendeurs v
JOIN total_ventes t ON v.id = t.vendeur_id
WHERE t.ca_total > 10000;

10. Fonctions de Fenêtrage (OVER)

Calculer sur un groupe de lignes sans les regrouper (conserve le détail).

fonction() OVER(PARTITION BY col ORDER BY col)
ROW_NUMBER()Numéro de ligne unique.
RANK()Classement (ex-aequo sautent).
DENSE_RANK()Classement (ex-aequo se suivent).
SUM(sal) OVER()Cumul progressif ou total fixe.
-- Exemple : Rang de salaire par département
SELECT nom, sal, dept,
RANK() OVER(PARTITION BY dept ORDER BY sal DESC) as rang
FROM employes;

11. Logique CASE WHEN

Équivalent de SI/ALORS.

SELECT nom,
CASE 
  WHEN age < 18 THEN 'Mineur'
  WHEN age < 65 THEN 'Adulte'
  ELSE 'Senior'
END AS categorie
FROM personnes;

12. Fonctions Utiles & Dates

Manipulation de Texte :
  • LOWER(s) / UPPER(s) : Change la casse.
  • TRIM(s) / LTRIM / RTRIM : Supprime les espaces.
  • LENGTH(s) : Nombre de caractères.
  • SUBSTR(s, start, len) : Extraction (1-indexed).
  • REPLACE(s, old, new) : Remplace une chaîne.
  • INSTR(s, sub) : Position d'une sous-chaîne.
Mathématiques :
  • ROUND(n, d) : Arrondi à d décimales.
  • ABS(n) : Valeur absolue.
  • FLOOR(n) / CEIL(n) : Arrondi inf/sup.
  • RANDOM() : Valeur aléatoire.
Gestion des Dates (SQLite/Standard) :
  • DATE('now') / TIME('now') / DATETIME('now')
  • STRFTIME('%Y-%m-%d %H:%M', date) : Formatage complet.
    %Y: Année, %m: Mois, %d: Jour, %H: Heure, %M: Min
  • julianday(d2) - julianday(d1) : Écart en jours décimaux.
  • DATE(col, '+1 month') : Ajout d'intervalle.
Conversion & Type :
  • CAST(val AS type) : ex: CAST(annee AS INT).
  • TYPEOF(col) : Vérifie le type de donnée stocké.
