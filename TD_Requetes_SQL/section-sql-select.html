Niveau 1 — SELECT de base

Fondamental ⏱ 30–40 min
Objectifs
  • Sélectionner toutes les colonnes ou un sous-ensemble avec SELECT et SELECT *
  • Filtrer des lignes avec WHERE et ses opérateurs de comparaison
  • Trier les résultats avec ORDER BY (ASC / DESC) et paginer avec LIMIT
  • Éliminer les doublons avec DISTINCT
  • Combiner plusieurs conditions avec AND, OR et IN
  • Comprendre le comportement de NULL dans les filtres

Rappel de cours — Syntaxe générale SELECT

Une requête SQL de sélection suit toujours cet ordre obligatoire de clauses :

SELECT  colonnes | *
FROM    table
WHERE   condition
ORDER BY colonne [ASC | DESC]
LIMIT   n;

Seules SELECT et FROM sont obligatoires. Les autres clauses sont optionnelles mais doivent respecter cet ordre si elles sont présentes.

NULL vs chaîne vide '' — NULL signifie valeur inconnue ou absente. Une chaîne vide '' est une valeur définie, elle vaut zéro caractère. Pour tester NULL, on n'écrit jamais = NULL — on écrit toujours IS NULL ou IS NOT NULL. En effet, NULL = NULL retourne NULL (inconnu), pas TRUE.

Q1 Fondamental

Listez tous les employés avec toutes leurs colonnes.

Rappel : Le joker * dans SELECT sélectionne l'ensemble des colonnes de la table. C'est pratique pour explorer une table, mais en production on préfère nommer explicitement les colonnes.
Ctrl+Entrée pour exécuter

Solution — Q1

SELECT * FROM employes;

SELECT * retourne toutes les colonnes dans l'ordre où elles ont été définies à la création de la table. Vous devriez obtenir 15 lignes avec les colonnes id, nom, prenom, salaire, departement_id, date_embauche et poste.

Q2 Fondamental

Listez uniquement le nom, le prénom et le salaire de tous les employés, triés par salaire décroissant (du plus élevé au plus bas).

Piège courant : Par défaut, ORDER BY trie dans l'ordre ascendant (ASC). Pour inverser l'ordre, il faut explicitement écrire DESC après le nom de la colonne.
Ctrl+Entrée pour exécuter

Solution — Q2

SELECT nom, prenom, salaire
FROM employes
ORDER BY salaire DESC;

On nomme explicitement les 3 colonnes souhaitées dans SELECT. ORDER BY salaire DESC trie du salaire le plus élevé (≈ 80 000 €) au plus bas (≈ 38 000 €). Sans DESC, l'ordre serait ascendant.

Q3 Fondamental

Listez tous les employés appartenant au département 2 (Ressources Humaines). Affichez toutes leurs colonnes.

Piège courant : En SQL, l'opérateur d'égalité est = (un seul signe égal). Le double == des langages de programmation comme Python ou JavaScript n'existe pas en SQL et provoquera une erreur de syntaxe.
Utilisez la clause WHERE avec la colonne departement_id et l'opérateur =. La valeur numérique 2 ne nécessite pas de guillemets.
Ctrl+Entrée pour exécuter

Solution — Q3

SELECT *
FROM employes
WHERE departement_id = 2;

La condition WHERE departement_id = 2 ne conserve que les lignes dont la clé étrangère pointe vers le département RH. Les entiers ne prennent pas de guillemets ; seules les chaînes de caractères en ont besoin ('texte').

Q4 Fondamental

Listez le nom, le prénom, le poste et le salaire de tous les employés dont le salaire est strictement supérieur à 50 000 €, triés par salaire décroissant.

Astuce : Les opérateurs de comparaison >, <, >=, <= fonctionnent aussi bien sur les nombres que sur les dates et les chaînes (ordre alphabétique pour les chaînes).
Utilisez WHERE salaire > 50000. N'oubliez pas d'ajouter ORDER BY salaire DESC pour trier.
Structure complète : SELECT nom, prenom, poste, salaire FROM employes WHERE salaire > 50000 ORDER BY salaire DESC;
Ctrl+Entrée pour exécuter

Solution — Q4

SELECT nom, prenom, poste, salaire
FROM employes
WHERE salaire > 50000
ORDER BY salaire DESC;

L'opérateur > exclut la valeur 50 000 elle-même. Pour inclure 50 000, on utiliserait >= 50000. Le tri final permet de lire immédiatement les profils les mieux rémunérés.

Q5 Intermédiaire

Listez tous les postes distincts existants dans l'entreprise (sans doublons), triés par ordre alphabétique.

À noter : DISTINCT s'applique à la combinaison de toutes les colonnes sélectionnées. Si vous écrivez SELECT DISTINCT poste, departement_id, deux lignes avec le même poste mais des départements différents seront toutes deux conservées.
Placez le mot-clé DISTINCT immédiatement après SELECT, avant le nom de la colonne : SELECT DISTINCT poste FROM …
Ctrl+Entrée pour exécuter

Solution — Q5

SELECT DISTINCT poste
FROM employes
ORDER BY poste;

DISTINCT élimine les valeurs en doublon. Sans lui, chaque ligne de la table serait retournée. L'ajout de ORDER BY poste produit une liste alphabétique facile à parcourir.

Q6 Intermédiaire

Affichez le nom, le prénom et le salaire des 3 employés les mieux payés de l'entreprise.

Piège courant : LIMIT doit toujours apparaître après ORDER BY. Écrire LIMIT 3 sans ORDER BY renvoie 3 lignes arbitraires — pas forcément les 3 meilleures.
Combinez ORDER BY salaire DESC pour trier du plus élevé au plus bas, puis LIMIT 3 pour ne garder que les 3 premières lignes.
Ctrl+Entrée pour exécuter

Solution — Q6

SELECT nom, prenom, salaire
FROM employes
ORDER BY salaire DESC
LIMIT 3;

L'ordre des clauses est obligatoire : SELECT → FROM → WHERE → ORDER BY → LIMIT. Ici WHERE est absent mais ORDER BY précède toujours LIMIT. Pour obtenir les rangs 4 à 6, on ajouterait OFFSET 3 après LIMIT 3.

Q7 Intermédiaire

Listez le nom, le prénom et la date d'embauche des employés embauchés après le 1er janvier 2020, triés par date d'embauche croissante (du plus récent au moins récent).

Format des dates en SQLite : Les dates sont stockées sous forme de chaîne au format 'YYYY-MM-DD' (ISO 8601). Les guillemets simples sont obligatoires. La comparaison lexicographique fonctionne correctement pour ce format.
Utilisez WHERE date_embauche > '2020-01-01'. N'oubliez pas les guillemets simples autour de la date.
Pour trier du plus ancien au plus récent, utilisez ORDER BY date_embauche ASC (ou simplement ORDER BY date_embauche car ASC est la valeur par défaut).
Ctrl+Entrée pour exécuter

Solution — Q7

SELECT nom, prenom, date_embauche
FROM employes
WHERE date_embauche > '2020-01-01'
ORDER BY date_embauche ASC;

L'opérateur > sur une chaîne ISO 8601 fonctionne comme une comparaison chronologique car le format YYYY-MM-DD est triable lexicographiquement. ASC est optionnel ici (c'est le comportement par défaut) mais le rendre explicite améliore la lisibilité.

Q8 Avancé

Listez le nom, le prénom, le poste et le salaire de tous les employés dont le département a pour localisation 'Paris' ou dont le département est Finance (id = 3).

Triez les résultats par département, puis par salaire décroissant.

Pourquoi une jointure ? La colonne localisation se trouve dans la table departements, pas dans employes. Pour filtrer les employés en fonction d'une propriété de leur département, il faut joindre les deux tables sur la clé étrangère employes.departement_id = departements.id.
Piège courant : Quand on joint deux tables, les colonnes de même nom (ici id, nom) deviennent ambiguës. Préfixez-les toujours avec l'alias de table : e.nom, d.nom.
Utilisez un JOIN entre employes e et departements d sur e.departement_id = d.id, puis filtrez avec WHERE d.localisation = 'Paris' OR e.departement_id = 3.
Une variante élégante : récupérez d'abord les id des départements concernés dans une sous-requête, puis utilisez WHERE e.departement_id IN (SELECT id FROM departements WHERE localisation = 'Paris' OR id = 3).
Ctrl+Entrée pour exécuter

Solution — Q8

Approche 1 Jointure + WHERE OR 

SELECT e.nom, e.prenom, e.poste, e.salaire, d.nom AS departement
FROM employes e
JOIN departements d ON e.departement_id = d.id
WHERE d.localisation = 'Paris'
   OR e.departement_id = 3
ORDER BY e.departement_id, e.salaire DESC;

Les alias e et d rendent les références de colonnes non ambiguës. d.nom AS departement renomme la colonne dans le résultat pour la distinguer de e.nom.

Approche 2 Sous-requête + IN 

SELECT nom, prenom, poste, salaire
FROM employes
WHERE departement_id IN (
    SELECT id
    FROM departements
    WHERE localisation = 'Paris'
       OR id = 3
)
ORDER BY departement_id, salaire DESC;

La sous-requête retourne la liste des id des départements satisfaisant la condition. IN (…) filtre ensuite les employés dont l'id de département figure dans cette liste. Les deux approches donnent le même résultat : les employés des départements Informatique (Paris), Finance (Paris/id=3).

Pour aller plus loin
  • Listez les employés dont le salaire est compris entre 45 000 et 65 000 € en utilisant BETWEEN 45000 AND 65000 — les bornes sont incluses.
  • Trouvez tous les employés occupant un poste contenant le mot "Senior" en utilisant LIKE '%Senior%' (le % représente n'importe quelle suite de caractères).
  • Triez les employés d'abord par département croissant, puis par salaire décroissant : ORDER BY departement_id ASC, salaire DESC — le tri multi-colonnes permet des classements composés.
  • Affichez les employés dont le departement_id est 1, 3 ou 4 en une seule condition avec WHERE departement_id IN (1, 3, 4) — plus lisible que trois conditions OR.
  • Trouvez les employés embauchés entre le 1er janvier 2018 et le 31 décembre 2021 en utilisant WHERE date_embauche BETWEEN '2018-01-01' AND '2021-12-31'.
