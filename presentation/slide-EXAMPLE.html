Pourquoi des bases de données ?

Avant les SGBD, les données étaient stockées dans des fichiers plats — redondance massive, incohérences, pas de contrôle des accès concurrents.

Un SGBD garantit les propriétés ACID : Atomicité, Cohérence, Isolation, Durabilité.
Piège courant : confondre un SGBD avec un simple fichier CSV ou Excel.

Modèle relationnel

Définition : Le modèle relationnel représente les données sous forme de relations (tables), composées de tuples (lignes) et d'attributs (colonnes).
  • Chaque table a une clé primaire unique par ligne
  • Les clés étrangères expriment les associations entre tables
  • Le schéma est défini avant l'insertion des données

SQL vs NoSQL

  • Schéma rigide et cohérent
  • Transactions ACID
  • Requêtes complexes (JOIN)
  • Scalabilité horizontale difficile
  • Migrations de schéma lourdes
  • Moins adapté aux données hétérogènes

Le choix dépend du volume, de la structure et des contraintes de cohérence.

Requête SQL — exemple

psql> -- Top produits, novembre 2024 SELECT p.nom, COUNT(c.id) AS nb_commandes, SUM(c.montant) AS total_ht FROM produits p JOIN commandes c ON c.produit_id = p.id WHERE c.date >= '2024-11-01' GROUP BY p.nom ORDER BY nb_commandes DESC LIMIT 10;
Toujours préférer JOIN explicite à une jointure dans le WHERE.
