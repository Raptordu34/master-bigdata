Objectifs

  • Créer une VM temporelle par semestre
  • Comparer les deux axes d'analyse

Prérequis : Q1.1 – Q1.3 — VM_Performance_Etudiants

Q1.4 Vue agrégée par département Intermédiaire

Créer une vue matérialisée VM_Stats_Departement qui affiche, pour chaque département : son nom, le nombre de cours proposés, le nombre d'enseignants et la moyenne des notes des étudiants.

Piège courant : Utiliser COUNT(a.ID_enseignant) sans DISTINCT. Un même enseignant peut intervenir dans plusieurs cours du même département : sans DISTINCT, on comptabiliserait les doublons.
Jointure nécessaire : Cours c LEFT JOIN Affectations a ON c.ID_cours = a.ID_cours LEFT JOIN Inscriptions i ON c.ID_cours = i.ID_cours. Le GROUP BY porte uniquement sur c.Departement.
Utilisez COUNT(DISTINCT c.ID_cours) et COUNT(DISTINCT a.ID_enseignant) pour compter les individus uniques et éviter les doublons dus aux jointures multiples.

Solution — Q1.4

CREATE MATERIALIZED VIEW VM_Stats_Departement AS
SELECT
    c.Departement,
    COUNT(DISTINCT c.ID_cours)          AS Nb_Cours,
    COUNT(DISTINCT a.ID_enseignant)     AS Nb_Enseignants,
    ROUND(AVG(i.Note), 2)               AS Moyenne_Etudiants
FROM   Cours c
LEFT JOIN Affectations a ON c.ID_cours = a.ID_cours
LEFT JOIN Inscriptions  i ON c.ID_cours = i.ID_cours
GROUP BY c.Departement;
Pourquoi COUNT DISTINCT ?
  • Un même cours peut avoir plusieurs affectations (enseignants différents sur plusieurs semestres). Sans DISTINCT, COUNT(a.ID_enseignant) compterait les doublons.
  • De même, un même enseignant peut intervenir dans plusieurs cours du même département : COUNT(DISTINCT a.ID_enseignant) donne le nombre d'individus uniques.

Exemple de résultat attendu :

Departement Nb_Cours Nb_Enseignants Moyenne_Etudiants
Informatique12713.42
Mathématiques8511.87
Physique6412.10
Attention à l'interprétation de la moyenne : La valeur calculée est la moyenne de toutes les notes de tous les cours du département, pondérée par le nombre d'inscriptions. Ce n'est pas la moyenne des moyennes par cours.
Q1.5 Vue temporelle par semestre Fondamental

Créer une vue matérialisée VM_Inscriptions_Semestre qui affiche, pour chaque semestre : le semestre, le nombre total d'inscriptions, le nombre d'étudiants distincts et la moyenne générale.

Piège courant : Confondre COUNT(*) et COUNT(DISTINCT ID_etudiant). Le premier compte les lignes d'inscription (un étudiant inscrit à 5 cours = 5 lignes) ; le second compte les étudiants uniques.
Source unique : table Inscriptions uniquement. Groupez par Semestre. Ajoutez un ORDER BY Semestre pour un affichage chronologique.

Solution — Q1.5

CREATE MATERIALIZED VIEW VM_Inscriptions_Semestre AS
SELECT
    Semestre,
    COUNT(*)                              AS Nb_Inscriptions,
    COUNT(DISTINCT ID_etudiant)           AS Nb_Etudiants_Distincts,
    ROUND(AVG(Note), 2)                   AS Moyenne_Generale
FROM   Inscriptions
GROUP BY Semestre
ORDER BY Semestre;

Exemple de résultat attendu :

Semestre Nb_Inscriptions Nb_Etudiants_Distincts Moyenne_Generale
2024-S14 8201 23012.74
2024-S25 1131 27513.01
2025-S15 3501 31012.55
2025-S24 9901 29813.18
Cas d'usage — tableau de bord de pilotage : VM_Inscriptions_Semestre est idéale pour alimenter un dashboard de suivi : évolution du nombre d'inscriptions d'un semestre à l'autre, tendance des résultats, etc. Elle remplace une agrégation coûteuse sur 500 000 lignes par une lecture de quelques dizaines de lignes pré-calculées.

Comparaison des deux vues

Critère VM_Stats_Departement VM_Inscriptions_Semestre
Axe d'analyse Organisation (département) Temporel (semestre)
Tables sources Cours, Affectations, Inscriptions (3 tables) Inscriptions uniquement (1 table)
Complexité de la VM Multi-JOIN + COUNT DISTINCT Simple GROUP BY
Nombre de lignes résultat = nombre de départements (~10–50) = nombre de semestres (~4–10)
Fréquence de rafraîchissement conseillée Hebdomadaire Quotidienne (fin de journée)
